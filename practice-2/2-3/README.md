Задание 2-3: Обработка сигналов<br>
Требования:<br>
•Используйте signal() или sigaction()<br>
•Обработчик должен быть async-signal-safe (используйте write() вместо printf())<br>
•Флаги: volatile sig_atomic_t<br>

Вариант 6: Родитель создаёт дочерний процесс. Дочерний засыпает на 30 секунд.<br>
Родитель через 5 секунд отправляет SIGUSR1. Дочерний обрабатывает и выводит<br>
"Signal received"<br>

Используемые функции:<br>
fork() - создание нового процесса<br>
signal() - установка обработчика сигнала<br>
kill() - отправка сигнала процессу<br>
wait() - ожидание завершения дочернего процесса<br>
sleep() - приостановка выполнения процесса<br>
write() - асинхронно-безопасный вывод <br>

Создает дочерний процесс с помощью fork()<br>
В ветке дочернего процесса:<br>
Устанавливает обработчик сигнала<br>
Засыпает на 30 секунд<br>
Проверяет флаг после пробуждения<br>
В ветке родительского процесса:<br>
Ждет 5 секунд<br>
Отправляет сигнал SIGUSR1<br>
Ожидает завершения дочернего процесса<br>
Для запуска используется ./task<br>

Пример вывода:<br>
Дочерний процесс создан (PID: 2911)<br>
Дочерний процесс засыпает<br>
Родительский процесс: sending SIGUSR1...<br>
Signal received<br>
Дочерний процесс получил сигнал<br>
Дочерний процесс завершился
